Теперь исправляем баг при котором при перемещении сайта с openserver в продакшен отключались подключенные расширения (типа поиска по имени и фильтра по весу). Плагин записывает в базу данных относительный путь до папки с расширением в виде хэша. При открытии сайта он проверяет какие расширения активированны перебирая все папки в папке с расширениями. Вычисляет для каждого пути свой хэш и сравнивает его с хэшами активированных расширений в базе.
Проблема в том что в опенсервере вычисляется хэш не относительного пути, а абсолютного, т.к. функция не может заменить начало абсолютного пути из-за обратных слешей, используемых в Windows.
Предлагаю исправить направление слэшей с \ на / изменив функцию в файле
wp-content/plugins/woocommerce-products-filter/classes/ext.php

Оригинал:
    //new from v.2.1.6
    public static function get_ext_idx($full_path)
    {
        return md5(str_replace(ABSPATH, '', $full_path));
    }
	
Исправленно:
    //new from v.2.1.6
    public static function get_ext_idx($full_path)
    {
        //Вот тут проблема определения активированных расширений при смене файлового пути до расширения. Замена не происходит т.к. в винде слеши в другую сторону
        $full_path_corrected = str_replace('\\', '/', $full_path);
        $ABSPATH_corrected = str_replace('\\', '/', ABSPATH);
        $f = str_replace($ABSPATH_corrected, '', $full_path_corrected);
        return md5($f);
    }
